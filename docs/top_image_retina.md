# トップ画像のRetinaディスプレイ対応

## 対象画像

トップ画像（`public/images/top.*`）

## 画像サイズ

CSS での表示サイズを基準に、1x / 2x / 3x の画像を用意する。

| 解像度 | サイズ      | ファイル名                   |
| ------ | ----------- | ---------------------------- |
| 1x     | 620 x 350   | `top.webp` / `top.jpg`       |
| 2x     | 1240 x 700  | `top@2x.webp` / `top@2x.jpg` |
| 3x     | 1860 x 1050 | `top@3x.webp` / `top@3x.jpg` |

1x のサイズはモバイル表示時の最大幅（767px × 100% × 80% ≒ 614px）を基準に、切りよく 620px とした。

## srcset / sizes の仕組み

### srcset の `w` 記述子

```html
srcset="images/top.webp 620w, images/top@2x.webp 1240w, images/top@3x.webp
1860w"
```

`w` は画像ファイルの実際のピクセル幅を表す。ブラウザに画像の横幅を伝える役割。

### sizes

```html
sizes="(max-width: 767px) 80vw, 30vw"
```

画像が画面上でどのくらいの幅で表示されるかをブラウザに伝える。

- `(max-width: 767px) 80vw` → モバイル: ビューポート幅の 80%
- `30vw` → デスクトップ: ビューポート幅の 30%（768px × 50% × 80% 相当）

### ブラウザの選択ロジック

ブラウザは `sizes` とデバイスのピクセル密度（DPR）を掛け合わせて、`srcset` から最適な画像を選択する。

**例: iPhone（375px 幅、DPR 3x）**

1. 表示幅 = 375 × 80% = 300px
2. 必要なピクセル = 300 × 3 = 900px
3. `srcset` から 900px 以上で最も近い `1240w` を選択

**例: デスクトップ（1440px 幅、DPR 1x）**

1. 表示幅 = 1440 × 30% = 432px
2. 必要なピクセル = 432 × 1 = 432px
3. `srcset` から `620w` を選択

### width / height 属性

```html
width="620" height="350"
```

レイアウトシフト（CLS）防止用。画像読み込み前にアスペクト比（620:350）を確保してスペースを予約する。実際の表示サイズは CSS で制御される。
